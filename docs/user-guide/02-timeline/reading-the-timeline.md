# Reading the Timeline

![Timeline overview](../../public/tutorial/2-visualiser.png)

The timeline is your primary planning canvas. It maps all initiatives across your portfolio onto a shared time axis so you can see how work is distributed across assets and time.

## Layout

The timeline is organised as a vertical stack of swimlanes. At the top level are asset categories (e.g. "Customer Identity", "Mobile"). Within each category are assets (e.g. "CIAM Platform"). Each asset has an initiative row where coloured bars represent work.

## The X-Axis

Time runs left to right. Each column represents a period — monthly columns by default, weekly columns in the 3-month view. Column headers show the month name or date.

## Bars

Each bar is one initiative. The width of a bar represents its duration. Its horizontal position reflects the start and end dates. Colour indicates the programme, strategy, or status depending on your current colour mode.

## Application Swimlane

If an asset has applications defined, an "Applications" swimlane appears below its initiative row. This swimlane displays lifecycle segment bars showing the current state of each application across time.

## Today Line

A red vertical line marks the current date. Use it as a quick reference to see what is happening now versus what is planned.

## Floating Legend

The floating legend is anchored to the bottom-right of the canvas. It shows what each colour means and the dependency arrow types. Click it to collapse or expand it.
