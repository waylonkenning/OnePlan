# Conflict Detection

![Conflict detection](../../public/features/conflict.png)

Conflict detection highlights scheduling problems directly on the timeline so you can address them without manually comparing dates.

## What It Detects

When two initiatives on the same asset overlap in time — meaning one starts before the other ends — a red "Conflict Detected" marker appears at the point of overlap.

## Boundary Rule

Initiatives that touch on the same date, where one ends exactly when the next begins, are not flagged as conflicts. Only genuine overlaps trigger the marker.

## Toggling

Use the conflict detection toggle button in the header to turn the markers on or off. The setting persists across reloads. On mobile, the toggle is in the settings sheet.

Conflict markers appear behind the sticky asset name labels when scrolling horizontally, so they do not obscure the row labels.
