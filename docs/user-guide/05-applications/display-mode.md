# Display Mode

![](../../public/features/display-mode.png)

The **display mode picker** in the header controls what layers the timeline shows. Use it to tailor the view to your audience or current task.

| Mode | What is shown |
|------|---------------|
| Both (default) | Initiative bars and the Applications swimlane together |
| Initiatives Only | Initiative bars only — the Applications swimlane is hidden |
| Applications Only | The Applications swimlane only — initiative bars are hidden |

## When to use each mode

- Use **Applications Only** when presenting the technology lifecycle to a stakeholder who does not need to see the initiative schedule. The view is cleaner and focuses attention on application statuses and durations.
- Use **Initiatives Only** to reduce visual complexity when the application layer is not relevant to the conversation.
- Leave it on **Both** for your day-to-day portfolio management work.

## Empty rows in Applications Only mode

In **Applications Only** mode, all asset rows are shown by default — even assets that have no applications configured yet. This lets you see the full shape of your portfolio and plan where lifecycle segments still need to be added.

If you want to hide assets that have no applications, set **Empty Rows** to **Hide** in the More Settings panel (the `···` button in the header). With that setting active, only assets with at least one application segment will be rendered.

## Persistence

The selected display mode persists across page reloads. You do not need to reset it each session.
