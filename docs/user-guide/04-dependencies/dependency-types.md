# Dependency Types

*Screenshot: all three arrow types visible side by side — to be captured.*

Scenia supports three dependency types. The type controls the arrow colour, arrowhead style, and the language used in the dependency report.

## Requires

A blue arrow with an arrowhead.

"A requires B to start first" — A cannot begin until B has finished.

Use this when work is genuinely blocked by another piece of work. This is the default type assigned when you draw a new dependency.

## Blocks

A red arrow with an arrowhead.

"A must finish before B can start" — A is blocking B.

The distinction from Requires is perspective: Blocks describes the constraint from the blocking initiative's point of view, rather than from the dependent initiative's point of view.

## Related

A dark arrow with no arrowhead.

A general informational connection. Neither initiative is strictly sequenced against the other.

Use this for work that is related or co-ordinated but not strictly dependent — for example, two streams that share a delivery window or a common stakeholder group.

## Changing a Dependency's Type

You can change a dependency's type at any time by clicking the arrow and editing in the panel. See [Editing Dependencies](editing-dependencies.md).

## Hiding Dependency Arrows

The dependency visibility can be toggled on and off via the Relationships toggle in the header. This affects all dependency arrows across the timeline without removing them.
