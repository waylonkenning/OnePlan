# Critical Path

*Screenshot: critical path highlighting active on the timeline — to be captured.*

## What It Is

The critical path is the longest chain of sequenced work across your dependency network — the sequence that determines the earliest possible completion date of the portfolio. Any delay to an initiative on the critical path delays the overall end date.

Understanding the critical path helps you prioritise risk management and focus attention on the work that actually constrains delivery.

## Enabling Critical Path Highlighting

Click the **Critical Path** toggle button in the header. Bars and arrows on the critical path are visually highlighted; non-critical items are dimmed. This makes the constraining sequence immediately visible across the timeline.

Click the button again to return to normal colouring.

## Persistence

The toggle state persists across page reloads. If you leave the critical path view active, it will still be active the next time you open the portfolio.

## Notes

- Critical path calculation is based on the dependency relationships you have drawn. If your dependency network is sparse, the critical path may simply be the longest single initiative with no dependencies connecting it to others.
- Adding or editing dependencies updates the critical path calculation immediately.
- Milestone dependencies are included in the critical path calculation.
