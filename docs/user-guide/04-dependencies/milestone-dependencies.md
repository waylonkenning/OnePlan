# Milestone Dependencies

![Milestone dependency](../../public/features/milestone-dependency.png)

A milestone can act as the source of a dependency. This is useful for modelling "this initiative cannot start until this milestone is reached" — for example, a go-live date, a regulatory approval, or a funding gate.

## How to Draw a Milestone Dependency

1. Hover over a milestone diamond on the timeline.
2. Drag vertically from the milestone toward an initiative bar.
3. Release over the target initiative bar to create the dependency.

## Behaviour

The resulting arrow works identically to a standard initiative-to-initiative dependency arrow. You can:

- Change its type (Requires, Blocks, Related)
- Reverse its direction
- Reposition it horizontally
- Delete it via the edit panel

## Reports

Milestone dependencies are included in the [Initiatives & Dependencies report](../09-reports/initiatives-dependencies-report.md) alongside standard dependencies.
